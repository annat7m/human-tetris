04

Decision log

Keep Recognition as a black box

Decided: Engine passes only mode; Recognition emits onShapeDetected and onPoint.

Why: It lets separate teams work without leaking camera logic into scoring or game state.

Considered: More detailed signals like attempts/confidence were explicitly excluded.

docs/interface-contract.md:1-12, 81-140

Use geometric heuristics, not training

Decided: Map arms to UP | OUT | DOWN | BENT, then table lookup to seven shapes.

Why: Hackathon speed: no ML training loop, transparent tuning knobs.

Considered: More robust ML was deferred; docs call L rotations risky/flaky.

docs/mediapipe-shape-recognition.md:13-38, 148-159; src/lib/pose-classifier.ts:1-42

One sensitivity dial for pointing

Decided: holdSeconds drives EMA and charge behavior while reach/drain constants stay internal.

Why: The pointing algorithm stays tunable without exposing a matrix of thresholds.

src/lib/pointing-recognition.ts:1-17, 66-84

Game judgement remains in the Engine

Decided: Matching, fallback random corner, scoring, clearing, and game over are handled in src/routes/index.tsx.

Why: Recognition never knows the target; that preserves the contract and makes callbacks deterministic inputs.

docs/interface-contract.md:86-140; src/routes/index.tsx:327-358

06

Mental model essentials

  • SHAPES is the source of truth for block geometry and has exactly seven IDs.
  • The engine phase is richer than Recognition mode: prep, matched, moving, and settle all map to Recognition idle.
  • Recognition must emit at most once per active phase; both shape and pointing detectors reset on mode changes.
  • MediaPipe coordinates use y-down screen space; mirroring is presentation-sensitive.
  • The board is 16×16 and placement scans from corners to stack pieces against existing blocks.
  • Shape classification is pure table lookup after arm-state reduction; tune constants before changing shape IDs.
  • Pointing chooses the most extreme reaching wrist, not merely the straightest arm.
  • Timing speeds up as lines are cleared, with minimum making/pointing durations.
  • TanStack Router file routes own pages; src/routes/__root.tsx owns document shell.
  • Vite skips Cloudflare plugin in test mode per project gotcha.
